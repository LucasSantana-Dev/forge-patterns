## üìù Usage Guidelines

### **For Forge Patterns Development**
1. **Apply Brainstorming**: Use @[/brainstorming] for pattern innovation and design
2. **Use Systematic Debugging**: Apply @[/systematic-debugging] for validation and troubleshooting
3. **Leverage MCP Knowledge**: Use @[/skill-mcp-docs-search] and @[/mcp-docs-and-tools]
4. **Focus on Cross-Project Standardization**: Ensure patterns work across all Forge projects

### **For Pattern Design**
1. **Explore Multiple Approaches**: Present 2-3 pattern options with trade-offs
2. **Validate Thoroughly**: Test patterns with real-world scenarios
3. **Document Comprehensively**: Provide clear implementation guidance
4. **Plan for Evolution**: Design patterns that can adapt and grow

### **For Security and Validation**
1. **Think Like an Attacker**: Consider all threat vectors and scenarios
2. **Implement Defense in Depth**: Use multiple layers of security controls
3. **Automate Everything**: Create automated validation and monitoring
4. **Plan for Incidents**: Design response and recovery procedures

### **For Automation and CI/CD**
1. **Start Small**: Begin with high-impact, low-risk automation
2. **Measure Everything**: Track automation effectiveness and ROI
3. **Iterate Continuously**: Refine automation based on usage and feedback
4. **Share Knowledge**: Document and share automation patterns

These enhanced prompts leverage brainstorming, systematic debugging, MCP expertise, and cross-project standardization to create comprehensive, effective patterns that elevate the entire Forge ecosystem through shared knowledge and automated best practices.
